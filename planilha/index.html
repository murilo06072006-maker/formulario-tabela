⌨️ Testes de Acessibilidade com Teclado

Como navegar nesta página apenas com o teclado:

Tab → Avançar para próximo elemento focável
Shift + Tab → Voltar ao elemento anterior
Enter → Confirmar / Submeter / Ativar botão
Space → Marcar checkbox / Acionar botão
↑ ↓ → Navegar em opções de radio e select
Esc → Cancelar / Fechar modal (se houver)
Tab na tabela → Focar células e botões de ação
Página abre → Tab revela o skip link
✅ O que foi implementado para acessibilidade nesta página:
  • Skip link → pressione Tab ao abrir a página para revelar o atalho "Pular para tabela"
  • :focus-visible → contorno azul visível em todos os elementos focáveis pelo teclado
  • aria-label em todos os botões de ação (Editar/Excluir) com nome do usuário
  • aria-required="true" em campos obrigatórios (complementa o atributo required)
  • aria-describedby vincula campos às suas dicas de ajuda para leitores de tela
  • scope="col" / scope="rowgroup" nos <th> para leitores de tela navegarem na tabela
  • role="search" no form de filtro → leitor de tela anuncia "região de pesquisa"
  • tabindex="0" no wrapper da tabela → tabela é focável por teclado no mobile
  • role="group" + aria-label nos grupos de radio e checkbox
